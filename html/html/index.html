HTML Reference

HTML

HTML(HyperText Markup Lauguage)은 웹 페이지를 만들기 위한 언어입니다.

  • HT = HyperText : 문서와 문서를 연결
  • M = Markup : 마크업, 태그
  • L = Launguage : 언어
  • HTML이란? 문서와 문서로 연결된 태그 언어를 말합니다.

태그(Tag)

태그는 정보를 정의하는 방식을 의미합니다.

  • 태그는 열린태그와 닫는태그가 있습니다.
  • 닫는 태그에는 "/"가 있어야 합니다.
  • 닫는 태그가 필요없는 것도 있습니다. ex) <br>, <hr>, <img>, <input>
  • HTML5에서는 "/"가 생략이 가능합니다.

블록 요소/인라인 요소(Block/Inline)

블록 요소는 박스, 인라인 요소는 텍스트를 의미합니다.

  • 블록 엘리먼트(Block Element)
    • 독립된 박스 영역으로, 한 줄에 하나에 블록 요소만 표현 할 수 있습니다.
    • 블록요소에는 블록요소, 인라인요소 둘 다를 포함 할 수 있습니다.
    • <div>,<p>,<h1>,<h6>,<ul>,<li>
  • 인라인 엘리먼드(Inline Element)
    • 독립된 텍스트 영역으로, 한 줄에 여러개의 인라인 요소를 표현 할 수 있습니다.
    • 인라인 요소에는 블록 요소가 포함 될 수 없지만, 인라인 요소는 포함 할 수 있습니다.
    • <span>,<a>,<strong>,<em>,<img>
    • <a>는 인라인 요소 이나 예외적으로 블록요소를 포함할 수 있습니다.


DTD(Document Type Declaration) 선언

웹 페이지를 제작하기 위해서는 웹 문서 형식 유형을 설정하여야 합니다.
  • 독타입을 설정하지 않거나 잘못 설정하면 브라우저에 따라 화면이 다르게 나오거나 렌더링이 달라지기 때문에 독타입을 설정해햐 합니다.
<!-- html5 -->
<!DOCTYPE html>

<!-- HTML 4.01 Strict -->
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd"> 

<!-- HTML 4.01 Transitional -->
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

<!-- HTML 4.01 Frameset -->
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" "http://www.w3.org/TR/html4/frameset.dtd">

<!-- XHTML 1.0 Strict -->
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

<!-- XHTML 1.0 Transitional -->
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

<!-- XHTML 1.0 Frameset -->
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Frameset//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-frameset.dtd">

언어 속성 설정

컨텐츠의 언어를 인식하고 언어 정보를 추출하는데 사용합니다. 미리 언어를 설정하면 효율설이 더 좋아집니다.
<!-- html5 -->
<html lang="ko">
  
<!-- html4 -->
<html lang="ko" xml:lang="ko" xmlns= "http://www.w3.org/1999/xhtml">

문서 정보 설정

웹 문서의 필요한 정보를 제공합니다.
<!-- html5 언어설정 -->
<meta charset="UTF-8">

<!-- html4 언어설정-->
<meta http-equiv="Content-Type" content="text/html; charset=utf-8">

<!-- 웹 문서에 대한 제작자 설정-->
<meta name="author" content="cone91118">

<!-- 웹 문서에 대한 설명 설정-->
<meta name="description" content="사이트 설명">

<!-- 웹 문서에 대한 키워드 설정-->
<meta name="keywords" content="키워드">

HTML Description

<html>

HTML 문서의 최상위 루트를 나타냅니다.

<head>

HTML 문서의 헤더 요소를 정의합니다.

<body>

HTML 문서의 본문에 해당하는 컨텐츠를 정의하는 영역입니다.

<meta>

문서에 대한 정보를 제공합니다.

<link>

문서를 외부 파일과 연결합니다.

<title>

문서의 제목을 설정합니다.

<h1> ~ <h6>

텍스트의 사이즈를 나타냅니다.

<p>

하나의 문단을 만들어줍니다.

<ul>

순서가 필요없는 문단을 만들어줍니다.(unordered list)

<ol>

숫자나 알파벳 등 순서가 있는 문단을 만들어줍니다.(ordered list)

<li>

목록을 만드는 태그 입니다. 단독으로 사용하지 않으며 <ul>,<ol>태그 내부에 들어갑니다.

<a>

하이퍼링크를 정의합니다.

  • href(hyper reference): 링크로 이동하는 경로를 설정합니다.
  • target: 새로운 브라우저를 선정합니다.
    • <a target_blank>: 링크 클릭시 새로운 브라우저 창으로 설정합니다.
    • <a target_parent>: 현재 브라우저의 부모 브라우저 창으로 설정합니다. 부모가 없는 경우 _self와 동일합니다.
    • <a target_self>: 현재와 동일한 브라우저 창으로 설정합니다.
    • <a target_top>: 최상위 브라우저 창에서 설정합니다. 부모가 없는 경우 _self와 동일합니다.
  • 방문하지 않은 링크는 밑줄과 파란색으로 표시됩니다.
  • 방문한 링크는 밑줄과 보라색으로 표시됩니다.
  • 활성화된 링크는 밑줄과 빨간색으로 표시됩니다.
  • <a> 태그는 예외적으로 HTML5에서 블록요소를 포함할 수 있습니다.

<strong>

글자를 굵게 표시하여, 텍스트를 종요하게 보이고자 할 때 사용합니다.

  • 웹 디자인과 웹 퍼블리싱

<em>

emphasized text 즉 특정 텍스트를 강조하고자 할 때 사용합니다.

<span>

인라인 범위(inline)을 표시합니다. 이 요소는 텍스트를 묶기 위한 범용적인 텍스트 컨테이너로써 텍스트의 시작과 종료 지점을 표시하여 인라인 텍스트를 묶어줍니다.

  • 텍스트 표현시 문장 단위(줄 단위)로 정합니다
  • inline 속성을 가지기 때문에 width, height 크기 지정을 할 수 없습니다.
  • 참고: div, span태그 사용법과 차이

<div>

텍스트를 묶기 위한 범용적인 텍스트 컨테이너로써 텍스트의 시작과 종료 지점을 표시하여 텍스트를 묶어줍니다.

  • 텍스트 표현시 사각형 박스로 구역을 정합니다
  • width, height 크기 지정이 가능합니다

<address>

HTML5에서 재정의된 요소로서, 저작자 또는 문서에 관한 연락처 정보를 표시합니다.

  • 문서의 저작자(author), 제공자(owner), 연락 가능한 이메일 주소, 회사 주소, 전화번호 등을 나열합니다
  • 현재 문서와 관련된 연락 정보에만 사용할 수 있습니다
  • 특정 섹션에서 사용되면 특정 섹션에 국한되고 푸터에서 사용되면 문서 전체에 대한 연락처 정보입니다
  • 기존과 다르게 마다 각 주소를 가질수 있게 하나의 문서에 여러개의 연락처가 들어갈 수 있게 바뀌었습니다
  • 블록 레벨 요소를 포함할 수 없으며 인라인 요소와 텍스트만 포함할 수 있습니다
  • 브라우저 대부분이 address 요소로 마크업된 텍스트를 이탤릭체로 표시하는 동시에 줄 띄기(line spacing)나 줄바꿈(line breaks)을 통해 다른 정보와 구분되도록 합니다

<hr/>

문서에서 문단을 의미적으로 분리하고자 할 때 사용합니다. (닫는태그X)

  • width : 수평선의 길이 값을 지정
  • color : 선의 색상 지정
  • noshade : 입체감을 없애서 평평하게 보이도록 만듬

기본적인 수평선 긋기. 단락 구분

<hr width="500px" color="green" noshade/>

<br>

line break = 텍스트에서 줄바꿈을 생성합니다.

  • 닫는태그가 없습니다.

<table>

표를 만드는 태그입니다.

  • 각각 셀로 만들어 지며 header cell과 standcell로 나누어 집니다.
  • <th> : table head, 표의 제목(라인의 헤드)을 쓰는 역할을 합니다. (기본값: 굵은 글씨체, 중앙 정렬)
  • <tr> : table row, 가로줄을 만드는 역할을 합니다. (기본값: 보통 글씨체, 왼쪽 정렬)
  • <td> : table data, 셀을 만드는 역할을 합니다. (기본값: 보통 글씨체, 왼쪽 정렬)

summary 속성
  • 테이블 요약을 지정. 테이블 요약은 복잡한 구성의 테이블에 대한 설명 정보를 의미
  • HTML5에서는 권장되지 않는 요소
  • 문서 상에 나타나지 않고 스크린리더와 같이 해당 정보를 필요로 하는 사용자 도구에 의해서만 출력됨
  • summary 속성이 테이블의 caption 요소보다 먼저 등장하기 때문에 혼란을 야기

border 속성

  • 테이블 테두리선을 지정

<input>

form 태그 안에서 추가되는 입력 요소들 중 가장 중요한 태그입니다.
input 태그는 사용자로 부터 정보를 받아들이는 용로도 사용되며
type속성을 이용해 입력 양식을 여러가지로 변경하여 사용합니다.

  • 로그인폼, 회원가입폼 등 여러가지 정보를 입력하는데 많이 사용됩니다.

  • 닫는태그가 없습니다.
  • input태그의 속성
    • type : 입력 태그의 유형
    • value : 입력 태그의 초기값을 말하며 사용자가 변경 가능합니다.
    • name : 서버로 전달되는 이름을 말합니다. (사용자 임의 지정)
    • pattern : 허용하는 입력의 형태를 정규식으로 지정합니다.
    • required : 입력양식 제출전에 반드시 채워져 있어야 함을 표현합니다.
    • readonly : 읽기 전용 필드
    • placeholder : 입력 힌트를 희미하게 보여줍니다.
    • autofocus : 페이지 로드시 자동으로 입력 포커스를 갖춥니다.
    • autocomplete : 자동으로 입력을 완성 합니다.
  • input태그 type속성 *참고사이트
    • text : 텍스트를 입력하는 창을 생성
    • password : 비밀번호를 입력하는 창을 생성
    • radio : 라디오 버튼을 생성
    • checkbox : 체크박스를 생성
    • file : 파일 이름을 입력하는 창을 생성
    • image : 이미지를 전송 버튼으로 만듭니다
    • hidden : 사용자에게 보이지는 않지만 서버로 전송됩니다.
    • submit : 제출/전송하는 버튼을 만듭니다
    • button : 일반 버튼을 생성
    • data : 날짜를 입력하는 컨트롤 창 생성
    • datatime : UTC 날짜와 시간 형식을 이용한 컨트롤 창 생성
    • datatime-local : 현지 날짜/시간
    • month : 월 / 년도
    • time : 시간
    • color : 색상 코드 입력 컨트롤 창 생성
    • week : 주와 연도 선택 컨트롤 창 생성
    • email : 표준 이메일 주소를 입력받아서 검증하는 컨트롤 창 생성
    • tel : 전화번호를 입력받아서 검증하는 컨트롤 창 생성
    • search : 검색어 입력 양식 창 생성
    • reange : 2개의 숫자 사이의 숫자 선택 슬라이더 컨트롤 창 생성
    • number : 숫자만 입력받는 컨트롤 창 생성
    • url : URL만 입력 받는 컨트롤 창 생성

<form>

*참고

폼(FORM)의 범위를 표시합니다. 폼은 사용자 입력을 위한 다양한 형식의 컨트롤(W3C는 입력필드, 버튼 등 폼을 구성하는 입력 요소를 컨트롤이라고 부름)로 구성되는 영역이며, 이 영역의 시작과 종료 지점은 form 요소에 의해 정의됩니다.

  • 폼 요소를 블록 레벨 요소이므로 문서 전체를 폼으로 지정하거나 한 문서 내에 여러 개의 폼을 지정할 수 있습니다.

  • form 속성
    • action : 실행 애플리케이션을 지정합니다. 실행 애플리케이션은 입력된 데이터를 처리하는 서버 또는 프로그램입니다. 이곳으로 데이터를 보내려면 브라우저가 전송 위치(URL)를 알아야 하며, 이를 위해 action 속성을 사용합니다.
    • accept-charset : 승인된 문자 세트(accepted character set encoding)을 지정합니다
    • enctype : 인코딩 타입(encoding type)을 지정합니다
      • enctype="application/x-www-form-urlencoded" : 서버에 보내기전에 모든 문자를 인코딩하는 방식이며 폼에 텍스트 데이터를 포함했을 때 지정합니다(기본값)
      • enctype="multipart/form-data" : 파일 업로드 컨트롤처럼 문자가 아닌 파일을 전송할 때 사용됩니다
      • enctype="text/plain" : 일반 텍스트로 인코딩됩니다
    • method : HTTP 메소드(HTTP method)를 지정합니다. HTTP 메소드는 클라이언트와 서버 간 데이터를 주고받기 위한 방식을 의미하는데 주로 사용되는 방식은 GET과 POST 방식입니다.
      • GET 방식 : 입력된 데이터가 URL에 의해 전송되므로 암호화하지 않으면 URL만으로도 어떤 데이터를 입력했는지 알 수 있습니다. POST 방식보다 상대적으로 보안이 취약하고 전송할 수 있는 데이터도 제한적 입니다.
      • POST 방식 : 입력된 데이터를 HTTP Body에 담아 전송하면서 서버측에서 데이터를 처리하는 방식입니다. HTTP Body는 URL에 비해 더 많은 정보를 담을 수 있기 때문에 POST 방식은 GET 방식보다 더 많은 양의 데이터를 전송할 수 있으며 상대적으로 보안이 높은 편입니다.
    • name : 폼 이름을 지정합니다
    • novalidate : 유효성 미확인을 지정합니다. 이 속성을 지정하면 폼에 입력된 데이터를 서버로 보낼 때 데이터를 체크하지 않습니다

<fieldset>

관련 있는 폼 필드 세트(form FIELD SET)를 표시합니다. 폼 필드 세트는 폼 내에서 관련 컨트롤을 하나의 그룹으로 묶은 것을 말합니다.

  • 폼을 효과적으로 계층화시킬 수 있습니다
  • legend 요소를 함께 사용해야 합니다

<legend>

fieldset 요소의 제목(LEGEND)을 표시합니다. fieldset 요소를 이용하여 여러 개의 컨트롤들을 묶었으면 이 묶음이 어떤 성격 또는 용도인지 알려줄 필요가 있으며, 이때 legend 요소를 사용합니다.

  • 이 요소를 사용하면 fieldset 요소로 묶인 영역 주변에 테두리 선이 나타납니다
  • 이 요소는 부모 요소인 fieldset 요소와 함께 사용해야 하므로 fieldset 요소의 첫 번째 자식 요소는 legend 요소가 되어야 합니다

section

일반적으로 문서의 콘텐츠 섹션을 의미합니다.
  • 콘텐츠와 관련된 한 가지 주제 영역을 말합니다.
  • section 요소는 문장이나 문서의 스타일링 요소가 아니기 때문에 편의나 영역을 위함이면 div 태그가 좋습니다.
  • section 요소는 제목이 없는 경우는 섹션이라고 할 수 없기 때문에 제목을 제공해야 합니다.
  • section 요소는 일반적인 주제가 아니라면 구체적인 요소(article, aside, nav)를 사용하는 것이 적절합니다.

article

콘텐츠의 독립적인 항목을 나타내는 콘텐츠를 의미합니다.
  • article은 포럼, 신문사, 잡지, 블러그 항목, 게시판 글 등은 콘텐츠의 독립적인 항목을 나타냅니다.
  • section은 하나의 주제를 나타낸다면, article은 주제를 묶은 독립적인 콘텐츠입니다.
  • section 요소 안에는 article 요소를 쓸 수 있으며, article 요소 안에도 section요소를 쓸 수 있습니다.

nav

페이지 내에서 이동할 수 있는 네비게이션 링크 그룹입니다.
  • nav는 문서의 핵심적인 페이지의 메뉴 및 서브메뉴에서 사용하고, 문서에서 주로 한 번 사용합니다.
  • 문서 안에 링크가 포함된 콘텐츠는 nav를 사용하지 않습니다.
  • nav는 핵심적인 네비게이션에 사용해야 하므로, footer 내에 링크 그룹의 사용은 적절하지 않습니다.

main

웹 문서의 주요 콘텐츠 영역을 나타낼 때 사용합니다.
  • main은 웹 페이지에서 한 번만 사용할 수 있으며, 접근성과 검색 영역에 노출을 향상시킵니다.
  • article, aside, footer, header, nav를 하위 요소로 사용 할 수 있습니다.

aside

웹 문서의 메인 콘텐츠와 관련된 사이트 콘텐츠 영역을 나타냅니다.
  • aside는 메인 콘텐츠와 관련된 사이트의 정보, 광고, 등 부분적인 정보를 그룹화 할 때 사용합니다.

header

웹 문서의 헤더 영역을 나타냅니다.
  • 헤더에는 웹 페이지에 대한 소개, 네비게이션 영역, 테이블 영역, 검색영역, 로고영역을 포함한 영역입니다.
  • 헤더에는 제목 태그가 포함될 수 있으며, 필수 조건은 아닙니다.
  • header는 section 콘텐츠가 아닌 그룹화하기 위한 요소이므로 section 요소를 포함할 수 없습니다.

footer

웹 문서의 footer영역을 나타냅니다.
  • 푸터는 저작권 정보, 회사 정보, 관련 링크, 주소, 바닥글, 사이트 정보 등을 포함하는 콘텐츠 영역입니다.
  • footer는 섹션 콘텐츠가 아닌 그룹을 나타내는 요소이며, section, article, aside 등을 포함할 수 있습니다.

HTML5 신규 태그*참고

  • 구조적인 요소 - 구역을 나누는 블록 요소인 div 요소를 대신한다.
    • header - 네비게이션을 돕거나 소개를 나타내는 그룹을 표현, 섹션의 차례나 검색폼 또는 관련된 로고등을 감싸는 용도로도 사용됨
    • nav - 다른 페이지나 같은 페이지 안에 다른 부분으로 이어주는 네비게이션 링크로 구성된 섹션을 표현
    • article - 문서내에서 독립적인 컨텐츠를 나타냄
    • section - 문서나 응용프로그램의 일반적인 섹션을 표현
    • aside - 페이지 전체 내용과는 어느정도 관련성이 있지만, 주요 내용과는 직접적인 연관성은 없는 분리된 내용을 담고 있음
    • footer - 주로 저작권 정보나 서비스 제공자 정보등을 나타냄
  • 멀티미디어 요소 - 플러그인 없이 멀티미디어를 재생할 수 있게 한다.
    • audio - 오디오 파일을 재생
    • video - 비디오나 영화를 재생
    • canvas - 스크립트와 해상도에 의존하는 비트맵 캔버스를 제공
  • command - 사용자가 실행할 수 있는 명령을 나타냄
  • datalist - 다른 컨트롤을 위해 미리 정의된 옵션의 집합을 나타냄
  • details - 추가적인 정보를 나타내거나 사용자가 요청하는 정보를 나타냄
  • embed - 브라우저에 설치되어 있는 플러그인이 필요한 외부 어플리케이션을 동작시킴
  • figure - 일러스트, 다이어그램, 사진, 코드등에 주석을 다는 용도로 사용됨
  • figcaption - 부모요소인 figure요소의 내용들에 대한 캡션, 혹은 제목을 나타냄

HTML5 요소 의미 변경 태그

  • em - 내용을 강조
  • hr - 문단레벨에서 주제의 분리를 나타냄

HTML5 제거된 태그

  • 비슷한 기능의 다른 태그로 대체 가능한 태그들 - acronym, applet, dir, isindex
  • 디자인적인 기능 이상의 의미가 없어 삭제 (CSS로 대체 가능한 태그들) - basefont, big, center, font, strike, tt
  • 웹 접근성 향상을 위해 사용을 지양하는 태그들 - frame, frameset, noframes

table의 활용

th scope table(col/row)

시작장애인들은 눈으로 보기 힘들기 때문에 웹페이지를 보려면 스크린리더기라는 프로그램을 사용하여 오직 귀로만 테이블 정보를 얻어야 하기 때문에, table에 scope="col"과 scope="row"를 작성하므로써 테이블의 정보를 인식하고 읽는 순서를 결정합니다.
  • scope는 th에 사용하는 속성으로, 제목과 내용을 연결해주는 기능을 갖고 있음.
    • ex1 ) 과일에(th) scope="col" 적용 : 과일이라는 제목셀부터 사과, 바나나 순으로 읽으라는 뜻
      • <table>
<colgroup><col><col><col><col></colgroup>
<thead>
<tr>
<th scope="col">과일</th>
<th scope="col">자동차</th>
<th scope="col">국가</th>
<th scope="col">가수</th>
</tr>
</thead>
<tbody>
<tr>
<td>사과</td>
<td>BMW</td>
<td>대한민국</td>
<td>마이클잭슨</td>
</tr>
<tr>
<td>바나나</td>
<td>아우디</td>
<td>미국</td>
<td>비욘세</td>
</tr>
</tbody>
</table>
    • ex2) 월요일(th)에 scope="col"을 적용 , 날씨(th)에 scope="row" - 월요일 -> 날씨 -> 맑음 으로 읽힘
