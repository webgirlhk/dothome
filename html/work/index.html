잡지 속 뷰티샷을 완성하는 포토샵 고급 기술


뷰티 아이템 촬영의 기초와 핵심

  • 뷰티 실버 제품 디자인 : 은색 빛반사, 둥근 원통형의 제품을 입체감있게 표현하기 위해 필라이팅,키라이팅
  • 유리병 향수 제품 디자인 : 텍스처, 내용물의 컬러감,유리 특유의 결,재질 등을 살리는 것이 중요함
  • 제품의 하이라이트와 쉐도우 부분의 디테일이 중요함
  • 창-정보-스포이드 클릭-잉크 총 사용량-잉크% 8~289%사이로 맞춰줘야 프린트했을때 결과가 좋음

럭셔리하고 몽환적인 분위기의 배경 만들기

  • 오른쪽 손가락 ->스포이드-> 곡선(톤 밝기 조정)
  • 마스크 -> 옵션 딜리트 -> 레이어마스크에 효과 > 부드러운 브러쉬 효과 / 불투명도적당히
  • 10~20% / 흐름(경계선) 폭을 넓게 그라데이션 추가 / 필요한 부분 터치

흐르는 방울 및 리퀴드 표현 및 합성 방법

  • 필터 -> 픽셀 유동화 : 뒤틀기도구 / 브러쉬크기로 농도 조절(폭 넓이) 50%
  • 압력 숫자 낮을수록 움직이는 폭 좁음, 제품과 구슬 사이즈에 맞게 방울합성
  • 끝부분은 레이어 마스크로 부드럽게 이어주기 (경도 0, 불투명도 30)
  • 클리핑 마스크로 색상균형(컬러밸런스), 노출(곡선)맞추기
  • 물방울 모양 블러도구로 경계선쪽을 부드럽게 만들기 (합성티 줄임)
  • 포커스 아웃된 뒤제품의 물방울은 가우시안흐림효과 적용후, 필요없는 부분 마스크로 지우기

어려워도 피할수 없는 리퀴드 촬영과 리터칭 기술

  • 선택영역지정> 이동/쉬프트/ 방향키로 이동, 편집 자유변형으로 팁 살리기
  • 모든 합성작업은 마스크를 사용함 / 브러쉬경도, 빠른 선택도구로 필요부분 선택
  • 빠른선택 : 퀵마스크 > 옵션 > 백스페이스
  • 선택>수정>매끄럽게 경계선 픽셀 단위를 부르럽게 만드는 기능임 (전체적으로 부드럽게1~3px정도)
  • 흐르는 메니큐어 뒤라인 겹치는 부분은 패스를이용해서 선택영역 (렌더링:0.8픽셀)만들고 마스크 씌우기
  • 흐르는 방울부분도 불투명도 조절 및 마스크로 끝부분 자연스럽게 연결

수분감 팡팡 터지는 이미지를 위한 물 모양 합성 방법

  • 먼저 베이스가 되는 제품 촬영이 완벽해야한다.
  • 라인툴을 이용해 합성 작업할 소스를 가져온다
  • 불투명도를 낮게 조절해 틀을맞춘다
  • 마스크를 활용해 필요없는 부분을 브러쉬 흐름 경도를 조절해 깔끔하게 지워준다.
  • 이때 상품 로고가 잘 보이게 합성해야 한다

생활코딩 - HTML


주요 태그


단락 - <p>

'paragraph=단락'의 줄임말로, 단락을 표현할 때 사용합니다.
  • 줄바꿈의 간격이 고정되어 있습니다.
  • 기본적으로 p태그의 디자인을 css 프로그래밍 언어를 통해서 디자인을 변경할 수 있습니다.

줄바꿈 - <br>

<br>태그는 'A forced line-break=강제로 줄을 바꾼다'의 줄임말입니다.
  • 'voil element'라고 해서 내용이 없는 태그, 열리는 태그만 사용합니다.
  • br태그를 써도 상관은 없지만 단락을 표현하는 거라면 p태그를 사용하는 것이 더 좋습니다.
  • br태그는 정보를 표현하는 것이 아니라, 시각적으로 줄바꿈을 할 뿐입니다.

이미지 - <img>

문서에 이미지를 포함합니다.
  • img태그는 닫히는 태그를 사용하지 않습니다.
  • 'src'속성을 사용해서 이미지의 경로, 이름을 적습니다.
  • 'width'속성을 사용해서 이미지의 크기를 변경하고 싶다면 값을 지정해줍니다.
    : height값은 알아서 지정해줍니다. 이미지를 왜곡시키지 않습니다.
  • 'alt(alternative text)'라는 속성을 사용해서 이미지의 설명을 적습니다.
    : 이미지가 깨졌을 때 alt 속성의 값(대체텍스트)을 화면에 표시해줍니다.
  • 'title'속성을 사용해서 이미지 설명을 적습니다.
    'tooltip'(마우스를 올려놨을 때 나오는 조그만 설명)을 통해서 도움말이 나오게 합니다. 
                            <img src="img.jpg" height="300" alt="산 이미지" title="산 이미지">

표 - <table>

웹페이지에 표를 삽입하는 태그입니다. (레이아웃을 만드는 과거의 방법)
  • <table> : tr,td태그 전체를 그룹핑해줍니다.
  • <tr> : table row(행)의 약자, tr태그로 같은 행에 속하는 td태그들을 그룹핑(grouping)을 시켜줍니다.
  • <td> : table data의 약자, 표에서 각각의 실제 데이터들을 의미, 항목 하나하나를 td태그로 감싸줌니다.
  • 표:table - 구조
    • 태그를 이용해서 정보들을 잘 구조화하고 표현해줄 필요가 있습니다.
    • <thead> : table의 제목 부분을 감싸줍니다
      : 제목이 되는 행은 td태그가 아니라 th태그로 정의합니다. 제목이 되는 데이터를 굵게 표시해줍니다.
    • <tbody> : table의 본문이 되는 부분을 감싸줍니다.
    • <tfoot> : 표에서 가장 아래쪽에 표시됩니다.
  • 표:table - 병합
    • 병합하고자하는 첫번째 부분에다가 속성을 적용합니다.
    • 중복되는 데이터는 삭제하면 됩니다.
    • rowspan="2" : 2개의 행을 병합해주는 속성
    • colspan="3" : 3개의 열을 병합해주는 속성

입력양식 - form

사용자가 입력한 정보를 서버로 전송할 때 사용하는 태그입니다.
  • 전체를 form태그로 감싸줍니다.
  • action="전송받는 서버url" 속성 : 전송받는 서버의 주소를 입력합니다.
  • <input> : 사용자로부터 정보를 입력받을 수 있는 태그입니다.
  • type="text" 속성 : 사용자로부터 문자 정보를 받을 수 있는 박스를 만듭니다.
  • type="password" 속성 : password에 해당되는 정보들이 **으로 표시됩니다.
  • type="submit" 속성 : 사용자가 입력한 정보가 서버로 전송되는 버튼을 만듭니다.
  • name="id" , name="pwd", name="address" 속성을 사용, 각각의 박스에 이름을 정해줍니다.
  • *name이라는 속성은 모든 폼태그에서 필요한 속성입니다.*
  • 각각의 이름의 값으로 서버에 데이터가 전송됩니다.

아이디 :

비밀번호 :

주소 : 

                            <form action="#">
                                <p>아이디 : <input type="text" name="id"></p>
                                <p>비밀번호 : <input type="password" name="pwd"></p>
                                <p>주소 : <input type="text" name="address"></p>
                                <input type="submit">
                            </form>

form - 텍스트 입력

  • <textarea> : 여러 줄을 입력 받을 때는 textarea태그를 사용해야 합니다.
    속성을 이용하여 textarea를 설정해줍니다.
    cols, rows 속성을 사용하여 입력칸의 박스 크기를 변경할 수 있습니다.

제한된 공간안에서 여러 개의 선택지를 선택할 수 있도록 하는 태그입니다.
  • 선택지들은 <option>로 감싸줍니다.
  • 선택을 할 수 있게 <option>로 감싼 선택지들을 <select>로 묶어줍니다.
  • <select name=""> name을 지정하여 서버에 전송 될 수 있게 적어줍니다.
  • <option value=""> value값을 지정하여 서버에 전송 될 수 있게 적어줍니다.
  • 다중선택 : <select>에 속성으로 multiple이라는 값을 넣어줍니다.
    값은 따로 없습니다. ctlr을 눌러야 복수선택이 가능합니다.

form - radio, checkbox

사용자가 여러개의 선택지 중에 하나 또는 복수개를 선택해서 제출할 수 있도록 하는 태그입니다.
  • radio 버튼 (단일선택)
    • <input type="radio"> 동그란 체크박스을 설정합니다.
    • 여러개를 만들었을 때는 name을 똑같이 지정해주면 같은 이름으로 지정되어 있는 것들끼리는 하나의 버튼만 선택됩니다.
    • 라디오 버튼은 네임값이 같은 것들 끼리 그룹핑이 되는 것이고
      같은 그룹에 속하는 것 중에서 하나가 선택되면 나머지의 선택이 해제되는 형태의 버튼입니다.
  • checkbox(다중선택)
    • 라디오 버튼과는 다르게 같은 name을 가지고 있는 여러개의 선택지들을 다중으로 선택할 수 있습니다.
    • 여러개를 만들어서 name을 똑같이 지정해주면 같은 이름으로 지정되어 있는 것들끼리 그룹핑 해줍니다.
    • 사용자가 웹페이지를 열었을 때 기본적으로 어떠한 항목이 선택되어 있게 하고 싶다면
      'checked'라고 하는 값이 없는 속성을 넣어줍니다.

form - 버튼

                            </form>

form - 컨트롤의 제목:label

  • label이 누구의 이름표인지를 표현해주기 위해서 'for'속성을 사용해야 합니다.
  • label 태그의 for속성의 이름과 = input 태그의 'id'속성의 이름이 같아야 합니다.

    표현방법이 다를 뿐 같은 결과로 나옵니다.

form - method

사용자가 입력한 데이터를 서버에 어떠한 방식으로 전송할 것인가 전송 방법의 타입을 지정합니다.
  • form태그에 속성으로 method="get or post"방식을 지정해줍니다.
  • get방식 : 기본값 , url을 통해서 데이터를 전송하는 방식
  • post방식 : url이 아니라 다른 방법으로 데이터를 숨겨서 전송하는 방식, 100% post값으로 전송하면 됩니다.
  • 방식의 결정권은 서버 엔지니어에게 있습니다.
  • 데이터를 전송시 기본적으로 method="post" 방식으로 지정해서 처리하면 됩니다.

의미론적 태그(semantic element)

HTML5에서는 웹페이지에서 통상 많이 사용하는 구조에 의미를 분명히 부여하기 위해서 새롭게 정의해서 제공하고 있습니다.
  • <article> : 웹페이지내에서 독립적인 콘텐츠를 나타냅니다. 하나의 문서 안에서 여러개의 article이 올 수 있습니다.
  • <aside> : 웹페이지 내에 주요 내용과는 직접적인 연관성은 없는 분리된 내용을 나타냅니다. 주로 사이드바의 형태
  • <details> : 추가적인 정보를 나타내거나 사용자가 요청하는 정보를 나타냅니다.
  • <figure>
    • 일러스트, 다이어그램, 사진, 코드등에 주석을 다는 용도로 사용됩니다.
    • 이 요소가 제거 되더라도 문서의 주된 흐름에는 큰 영향을 미치지 않습니다.
    • figure요소는 figcaption 요소를 포함할 수 있으며 figure요소안에서 맨앞이나 맨 마지막에만 올 수 있습니다.
    • 단, figcaption요소가 필수는 아닙니다.
  • <footer> : 사이트 하단에 위치하며, 저작권 정보나 서비스 제공자 정보 등을 나타냅니다.
    : 연락정보는 address요소에 담는것이 적당하며 footer요소안에 포함될 수도 있습니다.
  • <header>
    • 기능은 없지만, 웹페이지 상에서 가장 중요한 정보들이 표현되는 부분이라는 의미만 가지고 있는 태그입니다.
    • 일반적으로 섹션의 제목(h1 ~ h6요소 또는 hgroup요소)을 포함합니다. 필수X
    • 섹션의 차례나 검색폼 또는 관련된 로고 등을 감싸는 용도로도 사용될 수 있습니다.
  • <main> : 문서에서 가장 중심이 되는 컨텐츠를 정의합니다.
  • <mark>
    • 하나의 문서 내에서 다른 문맥과의 관련성을 나타내기 위해서 참조 목적으로 마킹되거나 하이라이트된 텍스트를 표현합니다.
    • 인용문의 특정 부분으로 주의를 끌기 위해 사용합니다.
    • 검색어와 매칭되는 부분을 하이라이팅하기 위해 사용합니다.
  • <nav>
    • 다른 페이지나 같은 페이지 안에 다른 부분으로 이어주는 네비게이션 링크로 구성된 섹션을 표현합니다.
    • nav요소는 주요 네비게이션 블럭을 구성하는 섹션입니다.
    • 페이지안에 모든 링크그룹이 nav요소일 필요는 없습니다.
    • 푸터에도 사이트의 중요한 곳을 링크하는 링크 그룹이 존재하지만, nav요소대신 footer요소로도 충분합니다.
  • <section>
    • 문서나 응용프로그램의 일반적인 섹션을 표현합니다.
    • 제목으로 시작하는 컨텐트를 의미적으로 그룹핑하기 위해 사용됩니다.
    • 홈페이지에서는 소개, 뉴스 아이템, 연락처등이 섹션으로 분리될 수 있습니다.
    • 요소의 컨텐츠를 배포 해도 이치에 맞다면 section요소 대신 article요소를 사용할 것을 권장합니다.
    • 요소가 단순히 스타일링을 위한 목적이나 스크립팅의 편의를 위해서 필요할 때 section요소보다는 div요소를 사용할 것을 권장합니다.
    • 딱히 정의하기 애매한 것들은 section 태그로 묶어줍니다.
  • <time>
    • 24시간에서의 시간 혹은 그레고리력에서의 정밀한 날짜를 나타냅니다.
    • 속성 - datetime : time요소가 지정하는 날짜나 시간을 나타냅니다.
    • 속성 - pubdate : 불리언 속성으로 부모요소중에 article요소나, 아티클 요소가 없다면 문서 전체가 발행된 날짜를 나타냅니다.
    • 현대의 날짜와 시간을 기계가 읽을 수 있는 방식으로 인코드해서 제공할 목적으로 사용합니다.

검색엔진 최적화 - 1

SEO(Search Engine Optimization) : 컨텐츠를 잘 해석할 수 있도록, 검색되었을 때 상위에 노출될 수 있도록 하는 것

html코드를 의미론적으로 타당한 태그로 설명하는 것이 검색엔진 최적화의 기본입니다.


검색엔진 최적화 - 2

  1. 명확하고 독창적인 타이틀의 사용
    • title 태그를 이용하여 컨텐츠를 대표하는 페이지 제목을 잘 작성해줍니다.
    • title 태그의 내용은 검색결과에 반영됩니다.
    • 페이지마다 고유한 title태그 작성합니다.
  2. description 메타 태그 활용하기
    • description 메타 태그안에 포함되어 있는 내용들은 검색할 때 중요한 검색대상이 됩니다.
    • description 메타 태그를 작성할 때는 1-2개의 문장이나 짧은 단락을 사용하는 것이 좋습니다.
  3. 페이지의 URL 구조를 개선하기
    • 이해하기 쉽고 어떤 정보를 담고 있는 url을 사용하는 것이 좋습니다.
    • 일반적인 페이지 이름, 긴 파일 이름, 반복적인 단어 사용 URL은 피해야 합니다.
  4. 사이트 내에서 이동하기 쉽게 만들기
    • 사이트 전체의 내용을 잘 구성하면서도 방문자가 사이트에서 현재 위치를 잘 알 수 있도록 해주는 디렉토리 구조를 사용합니다.
    • 특정 문서에 도달하기 위한 한 가지 형태의 URL 제공합니다.
    • 301 리디렉션, link rel=canonical href="오리지널 컨텐츠" 사용하여 해결합니다.
  5. 사이트 내에서 이동하기 쉽게 만들기
    • 크롤링 : 구글과 같은 검색엔진이 사이트의 내용을 가져가는 것
    • 홈페이지(=대문페이지)에 다른 웹페이지로 이동할 수 있는 링크들이 잘 작성되어 있어야 합니다.
    • 사용자가 URL의 일부를 제거하여 접속할 경우를 고려하여 상위 디렉토리로 이동시키는 방법이 있습니다.
    • 사이트의 이동은 자바스크립트보다는 하이퍼텍스트로 제공하는 것이 좋습니다.
  6. 우수한 품질의 콘텐츠와 서비스 제공
    • 검색 엔진을 위한 것이 아닌 사용자를 위한 콘텐츠를 작성합니다.
    • 사용자의 편의를 고려하지 않는 검색 엔진만을 위한 사이트는 오히려 검색 엔진으로부터 웹스팸으로 인정되어 차단될 수 있습니다.
  7. 보다 나은 앵커 텍스트 작성
    • 링크에 사용되는 앵커 텍스트는 링크되는 페이지가 무슨 내용인지에 대한 기본적인 정보를 제공해야 합니다.
    • “여기를 클릭”과 같은 일반적인 앵커 텍스트, 관련이 없는 앵커 텍스트 , 페이지의 URL을 대부분의 앵커 텍스트로 사용하는 것을 피해야 합니다.
    • 텍스트와 링크의 앵커 텍스트를 쉽게 구별할 수 있게 합니다.
  8. 이미지 사용의 최적화
    • 이미지 관련 정보는 “alt” 속성을 이용해 대체텍스트를 제공해야 합니다. (스크린 리더기를 사용하는 시각장애인들을 배려)
    • 이미지 파일은 'images'와 같은 일반적인 디렉토리에 저장하면 좋습니다.
    • 간결하면서도 설명적인 파일 이름과 alt 텍스트는 간단명료한 것이 가장 좋습니다.
  9. 제목 태그의 적절한 활용
    • 제목 태그를 활용하여 중요한 부분을 강조합니다.
    • 웹 페이지가 검색엔진에 잘 노출되고 적당한 콘텐츠가 필요로 하는 사용자에게 전달될 수 있는 가능성을 높여줍니다.
  10. robots.txt를 효과적으로 활용하기(웹페이지에 크롤링 하러 오는 로봇에 대응)
    • 검색에 노출이 필요하지 않은 부분을 robots.txt로 disallow합니다.
    • 민감한 콘텐츠는 암호화하거나 .htaccess를 사용하여 비밀번호로 보호하는 것이 보다 안전한 방법입니다.
  11. 페이지랭크
    • 어떤 사이트를 먼저 검색 결과에 노출해 주느냐 라고 하는 것의 기준
    • 많은 사이트들이 링크를 하고 있다면 그 사이트는 더 좋은 콘텐츠를 가질 확률이 높다라고 하는 전제
    • 페이지랭크가 높은 사이트가 링크를 하면 훨씬 더 페이지랭크가 높은 사이트가 됩니다.

언어란? 의사소통을 위한 약속 = JavaScript

  • JavaScript : 웹페이지를 동적으로, 프로그래밍적으로 제어하기 위해서 고안된 언어다.
  • 탈웹 : 자바스크립트 언어와 웹브라우저 환경을 분리해서 생각해야할 필요성들이 생기게 됨으로써, 최근에는 자바스크립트가 웹을 벗어나서 광범위하게 사용되고 있습니다. 대표적인 사례로 'google apps script'가 있습니다.

  • 브라우저 + 자바스크립트 코드를 작성할 수 있는 에디터

  • 맥은 텍스트에디트를 이용해서 파일을 만들면 됩니다.
  • 코드 <script>를 해석 할 때는 자바스크립트의 문법에 따라서 해석해서 실행하라고 알려주는 구문(태그)입니다.
  • alert('Hello world')는 경고창에 Hello world라는 문구를 출력하라는 일종의 명령입니다.

  • '개발도구 - sublime text'에 대한 수업을 따로 참고하세요.

숫자와 문자1 : 수의 표현

큰따옴표나 작은따옴표가 붙지 않은 숫자는 숫자로 인식합니다.
  • alert(1); = 경고창이 뜨면서 괄호안에 값이 표시됩니다.
  • alert(1+1); = 결과 : 2가 표시됩니다.
  • 곱하기를 할 때는 *(키보드 자판 숫자 8 위)를 사용합니다. alert(2 * 5); = 결과 : 10이 표시됩니다.
  • 나누기를 할 때는 /(슬래쉬, slash)를 사용합니다. alert(6 / 2); = 결과 : 3이 표시됩니다.
  • 자바스크립트에서는 사칙연산 보다 좀 더 복잡한 연산도 지원합니다.

숫자와 문자2 : 수의 연산

  • Math.pow(3,2); = 결과 : 9(3의 2승), pow=제곱
  • Math.round(10.6); 결과 : 11(10.6을 반올림), round=반올림
  • Math.ceil(10.2); = 결과 : 11(10.2를 올림), ceil=올림
  • Math.floor(10.6); = 결과 : 10(10.6을 내림), floor=내림
  • Math.sqrt(9); = 결과 : 3(9의 제곱근), sqrt=제곱근
  • Math.random(); = 결과 : 0부터 1.0 사이의 랜덤한 숫자
  • 100 * Math.random(); = 결과 : 100보다 작은 랜덤한 숫자

비교2 : ==과 ===

프로그래밍에서 비교란 주어진 값들이 같은지, 다른지, 큰지, 작은지를 구분하는 것을 의미한다.

== : 동등연산자(Equal Operator)는 좌항과 우항을 비교해서 서로 값이 같다면 true 다르다면 false가 됩니다.

<script>

=== : 일치연산자(Strict Equal Operator)는 좌항과 우항이 '정확'하게 같을 때 true 다르면 false가 됩니다.

<script>
alert(1=='1');   //true
alert(1==='1'); //false
</script>

서로 같은 수를 표현하고 있더라도 데이터 형이 같은 경우에만 같다고 판단하기 때문입니다. == 연산자 대신 === 연산자를 쓰는 것을 강력하게 권합니다.

  • undefined는 값이 정의되지 않은 상태, 프로그래머가 의도하지 않은 상황
<script>
alert(null == undefined);   //true
alert(null === undefined);  //false

alert(true === 1);  //false

alert(NaN === NaN); //false

true와 false는 블린(boolean)이라고 불리는 데이터 형식입니다.

NaN은 0/0과 같은 연산의 결과로 만들어지는 특수한 데이터 형인데 숫자가 아니라는 뜻입니다.

true와 false는 종료조건이 되는데, 이 값을 변경하는 것을 통해서 반복문을 종료시킬 수 있다.
반복문에서 종료조건을 잘못 지정하면 무한반복이 되거나, 반복문이 실행되지 않는다.
var i = 0;
// 종료조건으로 i의 값이 10보다 작다면 true, 같거나 크다면 false가 된다.
while(i < 10){
    // 반복이 실행될 때마다 coding everybody <br />이 출력된다. <br /> 줄바꿈을 의미하는 HTML 태그
    document.write('coding everybody <br />');
    // i의 값이 1씩 증가한다.
    i++
}

반복문3 : for문

//형식은 아래와 같다.
for(초기화; 반복조건; 반복이 될 때마다 실행되는 코드){
    반복해서 실행될 코드
}

//ex)다음예제를 보자
for(var i = 0; i < 10; i++){
    document.write('coding everybody'+i+'<br />');
}

for문은 제일 먼저 '초기화'를 한다.
위의 예제에서 초기화는 var i = 0;이다. 즉 변수 i의 값을 0으로 설정한 것이다.
그 다음에는 '반복조건'인 i < 10이 실행된다. 현재 i의 값은 0이다. 그렇기 때문에 이 조건은 참 중괄호 안의 내용이 실행된다.
i의 값이 0이기 때문에 'coding everybody0<br />'이라는 텍스트가 출력된다.
'반복해서 실행될 코드'의 실행이 끝나면 '반복이 될 때마다 실행되는 코드'가 실행된다.
i++는 현재 i의 값에 1을 더하라는 의미다. 현재 i의 값은 0이다. 따라서 i++의 결과로 i는 1이 되었다. 그리고 '반복조건'이 실행된다.
현재 i의 값은 1이기 때문에 i < 10은 참이다. 다시 '반복해서 실행될 코드'가 실행된다. 그렇게 반복해서 작업이 실행된다.
이 과정에서 i의 값은 반복 할 때마다 1씩 증가한다. 결국 i의 값이 10이 되는 순간 i < 10을 충족시키지 못하게 되고 반복문은 종료된다.

함수1 : 함수란?

함수(function)란 하나의 로직을 재실행 할 수 있도록 하는 것으로 코드의 '재사용성'을 높여줍니다.

함수의 형식

function 함수명( [인자...[,인자]] ){   //인자라는 값이 차례로 들어오는데 이 값은 함수를 호출할 때 함수의 로직으로 전달될 변수다. 인자는 생략 할 수 있다.
    코드   //함수를 호출 했을 때 실행하게 될 부분이 중괄호 안쪽에 온다.
    return 반환값
}

//예제
function numbering(){
    i = 0;
    while(i < 10){
        document.write(i);
        i += 1;
    }

함수2 : 함수의 효용

반복문은 일정한 반복을 '그 자리' 에서 실행할 때 의미가 있고, 함수는 일정한 반복을 '여러곳' 에서 실행할 때 의미가 있습니다.
  • 재사용성의 핵심은 어떠한 동일한 코드가 있을 때 이 코드를 여러곳에서 사용할 수 있는 형태로 만드는 것입니다.
  • 재사용성이 높아지면 함수를 사용하고 있는 여러 곳에서 유지보수가 용이, 코드의 가독성이 높아집니다.

함수3 : 출력

함수의 핵심은 입력과 출력입니다. 입력된 값을 연산해서 출력하는 것이 함수의 기본적인 역할입니다.

return

함수 내에서 사용한 return은 return 뒤에 따라오는 값을 함수의 결과로 반환하는 동시에 함수를 종료시킵니다.

//예제
function get_member(){
    return 'egoing';
    return 'k8805';
    return 'sorialgi';
}
alert(get_member());
  • 위의 예시에서 get_member를 출력(alert)한 결과는 'egoing'입니다.
  • 함수 내에서 문자열 'egoing'을 return한 후 함수가 종료되었기 때문에 'k8805'와 'sorialgi'는 출력하지 않습니다.

함수4 : 입력

인자(argument)는 함수로 유입되는 입력 값을 의미하는데, 어떤 값을 인자로 전달하느냐에 따라서 함수가 반환하는 값이나 메소드의 동작방법을 다르게 할 수 있습니다.

//예시
function get_argument(arg){ //정의된 (arg) 구문에 의해서 변수 arg의 값으로 숫자 1이 함수 안으로 전달됩니다.
    return arg;
}
alert(get_argument(1)); //1은 get_argument로 1이라는 값을 전달하겠다는 의미입니다.
alert(get_argument(2));

위의 예시에서 '1'은 인자(argument)이고, 'arg'는 매개변수(parameter)라고 부릅니다.

복수의 인자

여러개의 입력값을 받을 수 있지만, return값은 하나만 가질 수 있다는 한계가 있습니다.

//예제
function get_arguments(arg1, arg2){
    return arg1 + arg2
}
alert(get_arguments(10, 20));   //결과는 30입니다.
  • 함수를 호출 할 때 전달한 인자 10과 20은 함수의 선언부(1행)의 arg1, arg2에 차례로 할당됩니다.
  • 이렇게 전달된 값은 함수 내부로 전달되서 더해진 후에 반환됩니다.

배열1 : 배열의 문법

배열(array)이란 연관된 데이터를 모아서 통으로 관리하기 위해서 사용하는 데이터 타입입니다.

변수가 하나의 데이터를 저장하기 위한 것이라면 배열은 여러 개의 데이터를 하나의 변수에 저장하기 위한 것이라고 할 수 있습니다.

배열의 생성

  • 대괄호 안에 데이터를 콤마(,)로 구분해서 나열하면 배열이 됩니다.
  • var member = ['egoing', 'k8805', 'sorialgi'] = 각각의 데이터를 원소(Element)이라고 부릅니다.
var member = ['egoing', 'k8805', 'sorialgi']
alert(member[0]);
alert(member[1]);
alert(member[2]);
  • 결과는 egoing k8805 sorialgi 문자열을 차례로 경고창으로 출력 할 것입니다.
  • 즉 배열에 담겨있는 값을 가져올 때는 대괄호 안에 숫자를 넣습니다.
  • 이 숫자를 색인(index)라고 부르고 '0부터' 시작합니다.
  • 이 값을 이용해서 배열에 저정된 값을 가져올 수 있습니다.

배열2 : 배열의 효용

함수는 하나의 return값,출력만 가질 수 있기 때문에 이러한 한계를 극복할 수 있는 수단이 '배열'입니다.
  • return하는 결과 값을 배열로 정의 하면 하나의 값이 아니라 여러개의 값이 들어 갈 수 있습니다.
  • 배열은 연관되어있는 정보를 한꺼번에 다룰 수 있도록 해주기 때문에 프로그래밍을 하는데 있어서 아주 중요한 수단이 됩니다.

객체1 : 객체의 문법

객체(Object)는 배열과 같이 여러 개의 데이터를 하나의 변수에 저장하기 위한 것이고, 배열과의 차이점은 index의 값으로 숫자가 아니라 문자를 사용할 수 있습니다.

grades['sorialgi']
